# Demo Script (5-8 minutes)

1. Open frontend at `/` and show role-based login.
2. Login as `viewer`: show dashboard and charts, mention write actions are hidden.
3. Login as `analyst`: show records table and filtering by category/type.
4. Login as `admin`: create a new record and show summary changes.
5. In admin view, create a new user and explain status/role control.
6. Call `/audits` in Swagger with admin token and show write traceability.
7. Show `/docs` and mention migration + deployment files.
8. End with tests (`pytest -q`) as reliability proof.
